# JS Style Guide


### 1. Точка с запятой
>Точки с запятой должны присутствовать после каждого выражения, даже если их, казалось бы, можно пропустить.

```javascript
// плохо
alert('Привет') 

// хорошо
alert('Мир');
```

### 2. Комментарии
>Со временем программы становятся всё сложнее и сложнее. Возникает необходимость добавлять комментарии, которые бы описывали, что делает код и почему. 
Однострочные комментарии начинаются с двойной косой черты // 
Многострочные комментарии начинаются косой чертой со звёздочкой /* и заканчиваются звёздочкой с косой чертой */. 

```javascript
// плохо
alert('Мир'); /* Этот комментарий следует за инструкцией */

// Пример с двумя сообщениями. 
Это - многострочный комментарий.
alert('Привет'); 

// хорошо
alert('Мир');// Этот комментарий следует за инструкцией 

/* Пример с двумя сообщениями. 
Это - многострочный комментарий. */ 
alert('Привет'); 
```

### 3. Используйте понятные имена переменных и функций
>Код гораздо легче читать, когда при его написании используют понятные, описательные имена функций и переменных. 

```javascript
// плохо
function avg (a) {
	let s = a.reduce((x, y) => x + y)
	return s / a.length
}

// хорошо
function averageArray (array) {
	let sum = array.reduce((number, currentSum) => number + currentSum)
	return sum / array.length
}
```

### 4. Наименование переменных и функций
>Используйте Camel case («верблюжий регистр») для написания имен переменных и функций.

```javascript
// плохо
examplefunctionname

// хорошо
exampleFunctionName
```

### 5.Объявление переменных
>Вместо var используйте const — если пееменная постоянная, или let — если переменную необходимо переопределить.

```javascript
// плохо
var a = 'Привет';
var b = 1;

// хорошо
let a = 'Привет';
const = 2;
```

### 6. Горизонтальные оступы
>Для сохранения читаемости кода используйте горизонтаьные оступы

```javascript
// плохо
const users = [
{ name: "Bob", id: 1 },
{ name: "Jane", id: 2 },
{ name: "Fred", id: 3 }
];
const usernames = [];

// хорошо
const users = [
        { name: "Bob", id: 1 },
        { name: "Jane", id: 2 },
        { name: "Fred", id: 3 }
    ];
const usernames = [];
```

### 7. Строгое сравнение
>Использование обычного сравнения == может вызвать проблемы. Оператор строгого равенства === проверяет равенство без приведения типов.

```javascript
// плохо
alert( 0 == false ); // true 

// хорошо
alert( 0 === false ); // false, так как сравниваются разные типы 
```

### 8. Блоки кода
>Используйте фигурные скобки для кода, занимающего более одной строки


```javascript
// плохо
function sum() 
	return false;

// хорошо
function sum() {
	return false;
}
```

### 9.Объекты
>Для создания объекта используйте литеральную нотацию
 
```javascript
// плохо
const user = new Object();

// хорошо
const user = {};
```

### 10. Стрелочные функции
>Отдавайте предпочтение стрелочным функциям, они придают  краткость синтаксису

```javascript
// плохо
[1, 2, 3].map(function (x) {
  const y = x + 1;
  return x * y;
});

// хорошо
[1, 2, 3].map((x) => {
  const y = x + 1;
  return x * y;
});
```
