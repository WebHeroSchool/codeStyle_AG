# JS Best Practice Guide


### 1. Точка с запятой
>Точки с запятой должны присутствовать после каждого выражения, даже если их, казалось бы, можно пропустить. Это гарантирует более четкое разделение и стабильный код.

```javascript
// плохо
alert('Привет') 

// хорошо
alert('Мир');
```

### 2. Локальные переменные
>Сведите к минимуму использование глобальных переменных. Глобальные переменные и функции могут быть перезаписаны другими скриптами.

```javascript
// плохо
var a = 4;
function myFunction() {
  return a * a;
} 

// хорошо
function myFunction() {
  var a = 4;
  return a * a;
}
```

### 3. Используйте понятные имена переменных и функций
>Код гораздо легче читать, когда при его написании используют понятные, описательные имена функций и переменных. 

```javascript
// плохо
function avg (a) {
  let s = a.reduce((x, y) => x + y)
  return s / a.length
}

// хорошо
function averageArray (array) {
  let sum = array.reduce((number, currentSum) => number + currentSum)
  return sum / array.length
}
```

### 4. Наименование переменных и функций
>Используйте Camel case («верблюжий регистр») для написания имен переменных и функций, что делает более легким чтение строк.

```javascript
// плохо
examplefunctionname

// хорошо
exampleFunctionName
```

### 5.Объявление переменных
>Вместо var используйте const — если переменная постоянная, или let — если переменную необходимо переопределить. Объявление переменных без использования let и const автоматически делает их глобальными, что является плохой практикой при написании программного кода.

```javascript
// плохо
var a = 'Привет';
var b = 1;

// хорошо
let a = 'Привет';
const = 2;
```

### 6. Горизонтальные оступы
>Для сохранения читаемости кода используйте горизонтаьные оступы.

```javascript
// плохо
const users = [
{ name: "Bob", id: 1 },
{ name: "Jane", id: 2 },
{ name: "Fred", id: 3 }
];
const usernames = [];

// хорошо
const users = [
  { name: "Bob", id: 1 },
  { name: "Jane", id: 2 },
  { name: "Fred", id: 3 }
];
const usernames = [];
```

### 7. Строгое сравнение
>Рекомендуется всегда использовать строгое сравнение ===, это предотвращает ошибки приведения типов.

```javascript
// плохо
alert( 0 == false ); // true 

// хорошо
alert( 0 === false ); // false, так как сравниваются разные типы 
```

### 8. Никогда не объявляйте числовые, строковые или логические объекты
>Всегда обрабатывайте числа, строки или логические значения как примитивные значения. Не как объекты. Объявление этих типов как объектов снижает скорость выполнения.

```javascript
// плохо
let y = new String("John");

// хорошо
let x = "John";
```

### 9.Объекты
>Для создания объекта используйте литеральную нотацию.
 
```javascript
// плохо
const user = new Object();

// хорошо
const user = {};
```

### 10. Стрелочные функции
>Отдавайте предпочтение стрелочным функциям, они придают  краткость синтаксису.

```javascript
// плохо
[1, 2, 3].map(function (x) {
  const y = x + 1;
  return x * y;
});

// хорошо
[1, 2, 3].map((x) => {
  const y = x + 1;
  return x * y;
});
```
